# 分级钻取 + 海外止于国家级地图策略

新闻地区地图采用分级钻取：世界（国家色块）→ 中国可钻取到省、市级色块（DataV GeoJSON）；**海外止于国家级色块，其省/市仅作为文本元数据保留，不做逐级色块钻取、不做散点打点**。

这是真权衡的产物。均匀方案本应是「全球国家/省/市都可色块或散点钻取」，但它要求一份全球省/市的地理边界或坐标库——市级的全球 GeoJSON 与城市经纬度对照表体积巨大且无可靠单一来源，内置不现实、运行时聚合不可靠。分级方案牺牲海外的下钻粒度，换来：数据可行性（中国省市有成熟 DataV 数据，国家有 ECharts 世界图）+ 渲染简洁（纯 choropleth，无散点坐标依赖）+ AI 输出负担可控（海外城市只出名字不要求坐标）。

难逆转：DB schema（海外省/市无 adcode 列填充）、AI 提示词（不要求海外坐标）、前端树形 payload（海外国家节点无 provinces）三层共同编码此非对称性。无此背景，未来读者会困惑「为何海外新闻有省/市文字却地图上钻不进去、也没有散点」。

## Considered Options

- **均匀色块钻取**（全球国家/省/市皆可色块）：否决，全球市级 GeoJSON 无可靠来源。
- **海外城市散点（scatter）**：否决，需全球城市经纬度库，不存在可内置的版本。
- **分级 + 海外止于国家级（本决策）**：采纳。

## Consequences

- 海外新闻的省/市信息只出现在「点开该国后的新闻列表文字标签」里，地图本身不展示。
- 归一化对照表只覆盖中国省市 adcode + 国家 ISO，不含海外省市。
- 若未来获得可靠的全球省级 GeoJSON，可单独放开「海外省级色块」而无需改 AI 契约或 DB schema。
