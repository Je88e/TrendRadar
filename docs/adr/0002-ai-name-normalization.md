# AI 输出规范中文名，由 Python 归一化为地图 code

地区分类中，AI 只产出国家/省/市的**规范中文全称** + 层级 + 置信度；Python 用内置归一化对照表把这些中文名转成地图渲染所需的 code（中国省市 → adcode，国家 → ISO alpha-2 + ECharts 世界图英文名）。不让 AI 直接输出 adcode/ISO code。

真权衡的产物。备选是「AI 直出 adcode」省掉归一化层，但 LLM 不可靠地记忆行政区划码——错码会导致地图色块空白、且难排查。出中文名是 AI 的强项（自然、命中率高），代价是多一层归一化对照表 + 简称别名维护，以及偶尔的归一化失败（回退：照存名字、code 留 NULL、该地区不渲染色块但新闻仍进列表）。

难逆转：AI 提示词、DB schema（存中文 name 列 + code 列）、归一化模块三层共同编码此分工。无此背景，读者会疑惑「为何不让 AI 直接吐 code 省一层」。

## Considered Options

- **AI 直出 adcode/ISO**：否决，AI 记忆行政区划码不可靠，错码地图空白。
- **AI 出中文名 + Python 归一化（本决策）**：采纳。
- **AI 同时出名与 code 取 code**：否决，仍受 AI 错码污染，且增加 token。

## Consequences

- 提示词内嵌一份国家全称清单（~195），约束 AI 直出规范国名，提升归一化命中率；省市名不进提示词（340 个过多），靠内置别名表归一化。
- 归一化对照表是内置静态资产（仓库内 JSON），需随行政区划变更维护。
- 归一化失败为优雅降级，不阻断流程。
